## 发布规则

### 每日创作
- 目标：5-6章/天（随机）
- 字数：2500-3000字/章

### 每日发布（概率分布）
- **发布数量概率**：
  - 3章: 55% ⭐ 最常见
  - 2章: 30%
  - 4章: 12%
  - 5章: 3% 💎 偶尔爆发
- **计算方式**：`node scripts/get-publish-count.js`
- **硬性规则**：发布数量 ≤ (当前存量 + 新创作 - 2)

### 存量管理
- **最低存量**：1-2章（安全缓冲）
- **理想存量**：3-5章
- **最大存量**：10章（避免积压过多）

### 🎲 随机化要素
1. **创作数量**：5-6章随机
2. **发布数量**：2-5章按概率分布
3. **执行时间**：02:00-04:00随机（深夜创作）
4. **目的**：模拟真人深夜写作和发布行为

## 发布算法

```javascript
每晚 02:00-04:00 随机时间执行：

1. 创作数量 = random(5, 6)  // 随机 5 或 6 章
2. 创作 N 章新内容
3. 统计：未发布章节数 = 当前存量 + 新创作章节数
4. 目标发布数 = getPublishCount()  // 概率分布：3章55%、2章30%、4章12%、5章3%
5. 实际发布数 = min(目标发布数, 未发布章节数 - 2)  // 保留2章缓冲
6. 发布：从最早的未发布章节开始，发布 N 章
7. 记录：更新发布状态
8. 调整：下次执行时间随机化（明天 02:00-04:00）
```

**概率分布函数**：
```javascript
function getPublishCount() {
  const rand = Math.random() * 100;
  if (rand < 55) return 3;       // 55%
  else if (rand < 85) return 2;  // 30%
  else if (rand < 97) return 4;  // 12%
  else return 5;                 // 3%
}
```

## 示例场景

- 实际发布：min(3, 7+5-2) = 3章 ✅
- 剩余存量：9章

- 目标发布：2章（30%概率）
- 实际发布：min(2, 5+6-2) = 2章 ✅
- 剩余存量：9章

### 场景3：较多发布（4章，12%）
- 存量：6章
- 新创作：5章（随机）

- 实际发布：min(4, 6+5-2) = 4章 ✅

### 场景4：爆发发布（5章，3%）
- 存量：10章

- 存量：2章
- 新创作：5章（随机）

- 剩余存量：2章（触发最低保护）

## 紧急情况处理

### 存量耗尽（<2章）
- 暂停发布，专注创作
- 累积至少5章后恢复发布

### 存量积压（>10章）
- 临时增加发布量至8-10章/天
- 直到存量降至5章以下

## 发布时间策略

### 🌙 深夜随机发布（当前方案）
**目的**：模拟真实作者的深夜创作习惯

**执行时间**：每天 02:00 - 04:00 之间的随机时间
- 范围：120分钟窗口期
- 实现：每次任务执行完后，自动调整下次执行时间为明天的随机时间
- 脚本：`novel/scripts/update-next-run.js`

**任务流程**：
```
1. 随机时间点（例如 02:39）执行任务
2. 创作 5-6 章新内容（约 30-60 分钟）
3. 按概率分布发布 2-5 章到番茄小说（约 5-15 分钟）
4. 自动调整下次执行时间为明天 02:00-04:00 的随机时间
5. 通过飞书汇报结果（早上9:00汇报时包含）
```

**优势**：
- ✅ 深夜时段，符合兼职作者写作习惯
- ✅ 每天时间不固定，更像真人行为
- ✅ 避免白天高峰期，降低平台监控概率
- ✅ 发布数量概率分布，模拟创作状态波动
